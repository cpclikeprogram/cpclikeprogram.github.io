Set和Map

发表于 2022-08-05 | 分类于 interview

Set和Map的主要应用场景在与数组去重和数据存储。

Set是集合， Map是一种叫字典的数据结构。

  1. 集合：
    1. 特点：
      1. 集合是由一组无序且唯一（即不能重复）的项组成的， 可以想象成集合是一个既没有重复元素， 也没有顺序概念的数组。
      2. ES提供了新的数据结构Set， 它类似于数组， 但是成员的值都是唯一的， 没有重复的值
      3. Set本身是一个构造函数， 用来生成Set数据结构
  2. Set实例的属性和方法
    1. Set属性：
      1. size：返回集合所包含元素的数量
    2. Set的方法：
      1. 操作方法：
        1. add(value): 向集合添加一个新的项
        2. delete(value): 从集合中移出一个值
        3. has(value): 如果值在集合中存在， 返回true， 否则false
        4. clear(value): 移出集合里所有的项
      2. 遍历方法：
        1. Keys(): 返回一个包含集合中所有键的数组
        2. values(): 返回一个包含集合中所有值的数组
        3. entires: 返回一个包含集合中所有键值对的数组
        4. forEach(): 用于对集合成员执行某种操作，没有返回值
  3. 字典
    1. 字典与集合的共同点： 集合、字典可以存储不重复的值
    2. 不同点： 集合是以[val, val] 的形式存储元素， 字典是以[键，值]的形式存储
    3. 也就是说Map的主要用途也是用于存储数据， 相比于Object只提供”字符串-值“的对应， Map提供了“值—-值”的对应
    4. 属性
      1. size： 返回字典所包含的元素个数
    5. 操作方法：
      1. set(key, val): 向字典中添加新元素
      2. get(key): 通过键值查找特点的数值并返回
      3. has(key): 如果键存在字典中返回true， 否则false
      4. delete(key): 通过键值从字典中移出对应的数据
      5. clear(): 将这个字典中的所有元素删除
    6. 遍历方法：
      1. keys(): 将字典中包含的所有键名以数组形式返回
      2. values(): 将字典中包含的所有数值以数组形式返回
      3. forEach(): 遍历字典的所有成员
  4. Symbol： Symbol是ES6引入的一种原始数据类型， 表示独一无二的值， 当我们想为一个对象添加新方法， 通过创建要给属性名的Symbol类型的方法， 可以从根本上防止属性名冲突。
    1. 如何创建并使用Symbol?
    3. Symbol属性特性
      1. Symbol值不能与其他类型的值进行运算， 会报错
      2. Symbol值可以显式转为字符串、布尔值、但不能转为数值
    4. 通过Symbol函数创建
      1. Symbol函数前不能使用new命令， 因为生成的Symbol是一个原始类型的值， 不是对象
      2. 使用Symbol函数创建Symbol值时， 函数的参数只是表示对当前的Symbol值的描述， 这也就能够解释在创建时传入相同参数的Symbol函数的返回值是不相等的。
      3. 如果Symbol的参数是一个对象， 就会调用该对象的toString方法， 将其转为字符串， 然后才生成一个Symbol值。
    5. 使用SYmbol类型做对象属性名
      1. 使用Symbol值定义属性时， SYmbol值必须放在方括号中， 不能用点运算符， 因为点运算符后面后面总是字符串， 这会导致读取不到标识名。
      2. Symbol作为属性名、遍历对象的时候， 该属性不会出现在for…in、for…of循环中， 也不会被object.keys()、object.getOwnPropertyNames()、JSON.stringify()返回， 也不会被Object.prototype.hasOwnPrototype()访问到。
      3. 可读取到对象的Symbol属性的方法
        1. 有一个Object.getOwnPropertySymbols()方法， 可以获取指定对象的所有Symbol属性名， 该方法返回一个数组， 成员是当前对象的所有用作属性名的Symbol值。
        2. Reflect.ownKeys()方法可以返回所有类型的键名， 包括常规键名和Symbol键名。
    6. Symbol的一些方法
      1. [ES2019]提供了一个实例属性description， 直接返回Symbol的描述
      2. Symbol.for()方法可以重新使用同一个Symbol值。它接受一个字符串作为参数， 然后搜索有没有一个参数作为名称的Symbol值， 如果有， 就返回这个Symbol值， 否则就新建一个以该字符串为名称的Symbol值
      3. Symbol.for()和Symbol的区别在于前者会被登记在全局环境中共搜索， 后者不会
      4. Symbol.keyFor()方法返回一个已登记的Symbol类型值的key。
  5. BigInt: BigInt是一种新的数据类型， 用于当整数值大于Number数据类型支持的范围时。这种数据类型允许我们安全地对大整数执行算数操作， 表示高分辨率的时间戳， 使用大整数id， 等等，而不需要使用库。
    1. 为什么需要BigInt?
      1. 在JS中， 所有的数字都是以双精度64位浮点格式表示的，这会导致JS中的Number无法精确表示非常打的证书
    2. 如何创建并使用BigInt？
      1. 创建方法
        1. 直接在数字结尾追加n console.log(9007199254740995n );
        2. 用Bigint构造函数 BigInt(“9007199254740995”);

        2. BigInt不支持一元加号运算符， 这可能是某些程序可能依赖于+始终生成Number的不变量， 或者抛出异常。
        3. 因为隐式类型转换可能丢失信息， 所以不允许在bigInt和Number之间进行混合操作， 为混合使用大整数和浮点数时， 结果值可能无法由BigInt或Number精确表示。
        4. 不能将BigInt传递给Web api和内置的JS函数， 这些函数需要一个Number类型的数字。
        5. 当Boolean类型与BigInt类型相遇时， BigInt的处理方式与Number类似。
        6. BigInt可以正常地进行位运算， 如|、&、<<、>>和^
